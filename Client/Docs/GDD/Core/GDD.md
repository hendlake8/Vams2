# Vams2 - 게임 기획서 (Game Design Document)

## 1. 개요

### 한 줄 요약

판타지 세계관의 모바일 싱글 뱀서라이크. 5분 세션으로 스킬 빌드의 쾌감을 극대화한다.

### 장르

로그라이크 서바이벌 (Vampire Survivors-like)

### 플랫폼

모바일 (Android / iOS)

### 타겟 유저

- **주요 타겟**: 캐주얼~미드코어 모바일 유저
- **플레이 패턴**: 출퇴근, 점심시간, 취침 전 등 5분 단위의 짧은 세션 반복
- **동기**: 스킬 조합 실험, 대량 섬멸 쾌감, 장비 수집/강화 진행감

### 레퍼런스 및 차별점

**주요 레퍼런스**: 탕탕특공대 (Survivor.io)

| 항목 | 탕탕특공대 | Vams2 |
|------|-----------|-------|
| 온라인 | 협동, 랭킹, 시즌패스 | 완전 싱글 |
| 세션 길이 | 8분 / 15분 | 5분 |
| 테마 | 현대 도시 + 좀비 | 판타지 |
| 화면 방향 | 세로 (Portrait) | 세로 (Portrait) 고정 |
| 캐릭터 | 다수 (유료 해금) | MVP 1명 |
| 장비 | 초기부터 존재 | MVP 이후 추가 |

### 프로젝트 규모

- 1인 개발
- Unity 6 (6000.3.10f1) 기반
- MVP 핵심 기능 우선 개발

## 2. 핵심 메카닉

### 2.1 코어 루프

```
┌─────────────────────────────────────────────────────┐
│                    인게임 루프                        │
│                                                     │
│  이동(조이스틱) → 자동공격 → 적 처치 → 보석 흡수     │
│       ↑                                    ↓        │
│       │                              경험치 획득     │
│       │                                    ↓        │
│       └──────── 스킬 사용 ←── 레벨업 스킬선택(3택)   │
│                                    ↓                │
│                           [조건 충족 시 스킬 진화]    │
│                                                     │
│  ──── 5분 경과 or 사망 ────→ 세션 종료               │
└──────────────────────────────┬──────────────────────┘
                               ↓
┌──────────────────────────────────────────────────────┐
│                    메타 루프                           │
│                                                      │
│  결과 정산 → 재화 획득 → 로비 → 장비 강화/교체 → 재입장 │
└──────────────────────────────────────────────────────┘
```

### 2.2 이동 시스템

- **입력 방식**: 플로팅 싱글 조이스틱 (화면 터치 지점에서 생성)
- **이동 방향**: 8방향 + 아날로그 보간
- **이동 속도**: 기본값 5.0 units/sec (패시브/장비로 증감)
- **이동이 유일한 조작**: 공격, 스킬 발동은 모두 자동

### 2.3 자동공격 시스템

- **타겟팅**: 가장 가까운 적에게 자동 발사
- **기본 무기**: 매직 볼트 (직선 투사체, 단일 타겟)
- **공격 속도**: 기본 1.0초 간격 (패시브/장비로 감소)
- **데미지 계산**: `BaseDmg × (1 + SkillLv × 0.2) × 크리티컬 배율 × 장비 보정`

### 2.4 스킬 시스템

플레이어는 레벨업할 때마다 무작위 3개 스킬 중 1개를 선택한다.

**기본 규칙:**
- 액티브 스킬 최대 6슬롯, 패시브 스킬 최대 6슬롯
- 이미 보유한 스킬이 다시 제시되면 해당 스킬 레벨업 (최대 Lv 5)
- 빈 슬롯이 없으면 새 스킬 대신 기존 스킬 레벨업만 제시
- 선택지 3개 중 최소 1개는 미보유 스킬 (슬롯 여유가 있을 때)

**액티브 스킬 (6종):**

| 스킬 | 공격 패턴 | 특성 | Lv1 기본값 |
|------|-----------|------|-----------|
| 매직 볼트 | 직선 투사체 | 기본 무기 업그레이드형, 레벨업 시 투사체 수 증가 | DMG 10, 간격 1.0초 |
| 회전 검 | 캐릭터 주변 회전 | 근접 방어, 레벨업 시 검 수 증가 | DMG 15, 검 2개 |
| 파이어볼 | 폭발 범위 | 고데미지, 레벨업 시 폭발 반경 증가 | DMG 25, 반경 1.5 |
| 아이스 스피어 | 관통 직선 | 감속 30% 부여 (2초), 레벨업 시 감속률 증가 | DMG 12, 감속 30% |
| 라이트닝 | 체인 번개 | 다수 타겟, 레벨업 시 체인 수 증가 | DMG 18, 체인 3 |
| 포이즌 클라우드 | 장판 설치 | 지속 데미지, 레벨업 시 장판 크기 증가 | DMG 5/tick, 지속 3초 |

**패시브 스킬 (4종):**

| 스킬 | 효과 | Lv1 기본값 | Lv5 최대값 |
|------|------|-----------|-----------|
| 마나 서지 | 공격속도 증가 | +10% | +50% |
| 원소 핵 | 모든 원소(화/빙/뇌/독) 데미지 증가 | +15% | +75% |
| 마법 갑옷 | 방어력 증가 + 넉백 저항 | 방어 +5, 넉백 저항 20% | 방어 +25, 넉백 저항 100% |
| 신속의 부츠 | 이동속도 + 회피율 | 이속 +8%, 회피 3% | 이속 +40%, 회피 15% |

### 2.5 스킬 진화 (돌파)

특정 액티브 + 패시브 조합이 **모두 Lv 5(최대)**일 때 진화 가능.
레벨업 선택지에 진화 옵션이 등장하며, 선택 시 두 스킬이 합쳐져 1개의 진화 스킬로 변환된다.

| 액티브 | 패시브 | 진화 결과 | 진화 효과 |
|--------|--------|-----------|-----------|
| 파이어볼 | 원소 핵 | 메테오 스트라이크 | 하늘에서 운석 낙하, 광역 폭발. 기존 파이어볼 데미지 ×2, 폭발 반경 ×2 |
| 라이트닝 | 마나 서지 | 썬더 스톰 | 화면 전체에 번개 낙뢰. 체인 제한 없이 화면 내 전체 적 타격 |
| 회전 검 | 마법 갑옷 | 블레이드 배리어 | 검의 회전 반경 대폭 확대 + 접촉 시 적 넉백. 데미지 ×2 |

**진화 시 공통 규칙:**
- 데미지 2배 + 시각 이펙트 대폭 강화
- 진화 후 해당 액티브·패시브 슬롯 1개씩 회수 → 빈 슬롯에 새 스킬 장착 가능
- 진화 스킬은 추가 레벨업 불가 (최종 형태)

### 2.6 경험치 & 레벨업

- 적 처치 시 경험치 보석 드롭 (적에게 다가가면 자동 흡수)
- 흡수 범위: 기본 2.0 units (장비/패시브로 확장 가능)
- 레벨업 요구 경험치: `10 + (현재 레벨 × 5)`
  - Lv 1→2: 15, Lv 5→6: 35, Lv 10→11: 60
- 레벨업 시 게임 일시정지 → 스킬 3택 UI 표시

### 2.7 시스템 간 상호작용

```
[이동] ──→ 적 회피, 보석 흡수 범위 진입
  ↕
[자동공격] ──→ 적 처치 ──→ [보석 드롭]
  ↑                              ↓
[스킬 선택] ←── [레벨업] ←── [경험치 획득]
  ↓
[스킬 진화] ──→ 슬롯 회수 ──→ 추가 스킬 장착 가능
  ↕
[장비(메타)] ──→ 기본 스탯 보정 ──→ 인게임 전투력 변화
```

## 3. 게임 플로우

### 3.1 세션 흐름

```
앱 실행 → 타이틀 → 로비
                     ├── 장비 관리 (장착/강화/합성)
                     ├── 캐릭터 정보
                     └── 출격 버튼
                           ↓
                     인게임 (5분 세션)
                           ↓
                     결과 정산 화면
                     ├── 처치 수, 획득 재화, 도달 레벨 표시
                     └── 로비 복귀 버튼
                           ↓
                     로비 (반복)
```

**재시작 버튼 없음** — 반드시 로비를 경유하여 장비 상태를 확인하게 유도.

### 3.2 인게임 진행 구조 (MVP)

MVP에서는 **단일 맵 생존 모드**만 제공한다.

- 맵: 타일 기반 무한 반복 (판타지 숲/초원)
- 목표: 5분 생존 또는 최종 보스 처치
- 클리어 보상 > 사망 보상 (클리어 인센티브)

### 3.3 난이도 곡선 및 웨이브 페이싱 (5분 기준)

| 구간 | 시간 | 몬스터 | 설계 의도 | 예상 플레이어 레벨 |
|------|------|--------|-----------|-------------------|
| 온보딩 | 0:00~0:50 | 슬라임 (느림, 저체력) | 이동 조작 학습, 자동공격 체험 | Lv 1~3 |
| 회피 학습 | 0:50~1:30 | +늑대 (빠름) | 이동 회피의 중요성 체감 | Lv 3~5 |
| 미니보스 1 | 1:30 | 보스 아레나 생성 | 첫 보스 경험, 패턴 회피 | Lv 5~6 |
| 빌드 구간 | 1:30~2:30 | +스켈레톤 (원거리) | 스킬 선택이 생존에 영향을 주기 시작 | Lv 6~9 |
| 미니보스 2 | 2:30 | 보스 아레나 | 빌드 중간 점검 | Lv 9~10 |
| 밀도 급증 | 2:30~4:00 | 혼합 웨이브, 밀도 ×2 | 스킬 진화 발동 타이밍, 쾌감 극대화 구간 | Lv 10~15 |
| 미니보스 3 | 4:00 | 보스 아레나 | 완성된 빌드로 보스 처치 | Lv 15~17 |
| 최고 밀도 | 4:00~4:30 | +엘리트 몬스터 | 빌드 완성도 최종 시험 | Lv 17~19 |
| 최종 보스 | 4:30~5:00 | 최종 보스 아레나 | 클라이맥스, 클리어 판정 | Lv 19~20 |

**보스 아레나 규칙:**
- 보스 등장 시 화면 경계에 원형 벽 생성
- 보스 처치 전까지 일반 몬스터 스폰 중단 (보스 집중)
- 보스 처치 후 벽 해제, 보상 보석 대량 드롭

### 3.4 신규 시스템 해금 순서

추후 확정. MVP에서는 모든 시스템이 즉시 사용 가능.

## 4. 컨텐츠 설계

### 4.1 캐릭터 시스템

**MVP: 기본 캐릭터 1명**

| 항목 | 값 |
|------|-----|
| 이름 | 루드 |
| 설정 | 풋내기 마법사 |
| 기본 HP | 100 |
| 기본 공격력 | 10 |
| 이동 속도 | 5.0 units/sec |
| 크리티컬 확률 | 5% |
| 크리티컬 배율 | 1.5x |
| 기본 무기 | 매직 볼트 |

MVP 이후 다중 캐릭터 추가 시, 각 캐릭터는 고유 기본 무기와 스탯 차이를 가진다.

### 4.2 몬스터 설계

**일반 몬스터 (4종):**

| 몬스터 | 행동 패턴 | 기본 HP | 기본 DMG | 이속 | 드롭 EXP |
|--------|-----------|---------|---------|------|----------|
| 슬라임 | 직선 추적 (느림) | 10 | 5 | 2.0 | 1 |
| 늑대 | 직선 추적 (빠름) | 15 | 8 | 6.0 | 2 |
| 스켈레톤 | 원거리 (화살) + 느린 추적 | 20 | 10 (원거리) | 2.5 | 3 |
| 엘리트 슬라임 | 직선 추적 + 분열 (사망 시 소형 2마리) | 50 | 12 | 3.0 | 5 |

**보스 (미니보스 3종 + 최종보스 1종):**

- 일반 몬스터와의 핵심 차이: **원거리 공격**을 사용한다
- 보스 등장 시 아레나가 생성되며, 보스 처치 전까지 일반 몬스터 스폰 중단
- 구체적인 보스별 패턴/수치는 프로토타입 단계에서 확정

### 4.3 드롭 아이템 (인게임)

| 아이템 | 드롭 소스 | 효과 |
|--------|-----------|------|
| 경험치 보석 (소) | 일반 몬스터 | EXP 1~5 |
| 경험치 보석 (대) | 미니보스, 엘리트 | EXP 10~20 |
| 회복 고기 | 상자 파괴, 낮은 확률 드롭 | HP 20% 회복 |
| 자석 | 상자 파괴 | 화면 내 모든 보석 즉시 흡수 |
| 폭탄 | 상자 파괴 | 화면 내 모든 적에게 50 데미지 |
| 골드 | 모든 몬스터 (낮은 확률) | 로비 재화 |
| 장비 상자 | 보스 처치, 클리어 보상 | 랜덤 장비 1개 |

### 4.4 장비 시스템 (MVP 이후 1순위)

MVP 이후 첫 번째 확장으로 장비 시스템을 도입할 예정.
로비에서 장비를 장착/강화하여 인게임 스탯에 반영하는 메타 진행 루프를 추가한다.
구체적인 슬롯 구성, 등급 체계, 합성 규칙은 추후 확정.

## 5. 내러티브

> (브리프 미포함 — 검토 필요)
> MVP에서 내러티브는 최소한으로 구현한다. 아래는 세계관 뼈대 제안이다.

### 5.1 세계관

마법의 숲 '엘드라 그로브'에 갑작스러운 마수(魔獸)의 대침공이 시작되었다. 숲의 수호자인 플레이어 캐릭터가 밀려드는 마수들을 물리치고 숲을 지켜야 한다.

### 5.2 톤앤매너

- 밝고 경쾌한 판타지 (다크 판타지 아님)
- 텍스트 최소화 — 환경과 비주얼로 분위기 전달
- 심각한 서사보다는 "점점 강해지는 쾌감"에 집중

### 5.3 게임플레이와 서사의 통합

- 스테이지/챕터 확장 시 맵 테마로 세계관 확장 (숲 → 동굴 → 폐성 등)
- 보스 몬스터에 간단한 이름/설정 부여로 목표감 제공
- MVP 단계에서는 별도 스토리 컷신이나 대화 없음

## 6. 아트 방향

### 6.1 비주얼 스타일

- **시점**: 2D 탑다운
- **스타일**: 캐주얼 판타지 (귀엽거나 친근한 톤, 과도한 리얼리즘 지양)
- **화면 방향**: 세로 (Portrait) 고정
- **해상도 기준**: 1080×1920 기반 (9:16), 모바일 세로 종횡비 대응

### 6.2 에셋 전략 (1인 개발)

**핵심 원칙: 에셋 스토어 활용 우선**

1. Unity Asset Store에서 "2D Fantasy Top-down" 에셋 팩 1개를 기준 팩으로 선정
2. 기준 팩의 색감/해상도/스타일에 맞춰 나머지 에셋 선별
3. 직접 제작은 에셋이 없는 경우에만 최소한으로

**에셋 확보 우선순위:**

| 순위 | 에셋 종류 | 사유 |
|------|-----------|------|
| 1 | 캐릭터 스프라이트 (8방향 이동) | 가장 자주 보이는 에셋 |
| 2 | 몬스터 스프라이트 (4~5종) | 코어 루프에 필수 |
| 3 | 타일맵 (숲/초원) | 맵 구성 기본 |
| 4 | 이펙트/VFX (불, 번개, 얼음 등) | 스킬 피드백의 핵심 |
| 5 | UI 아이콘 (스킬, 장비) | 레벨업 UI에 필요 |

### 6.3 캐릭터 아트 방침

- 플레이어: 로브를 입은 풋내기 마법사 "루드"
- 적: 슬라임, 늑대, 스켈레톤 등 전형적 판타지 몬스터 (즉시 인지 가능)
- 보스: 일반 몬스터 대비 2~3배 크기, 색상 차별화

### 6.4 환경/배경 아트 방침

- MVP: 숲/초원 타일맵 1세트
- 타일 반복이 눈에 띄지 않도록 변형 타일 3~4종 혼용
- 장식 오브젝트 (나무, 바위, 꽃 등)를 랜덤 배치하여 단조로움 방지

### 6.5 UI 아트 방침

- 판타지 RPG 스타일 프레임 (양피지, 금속 테두리 등)
- 아이콘은 심볼 위주로 작은 화면에서도 식별 가능하게
- 색상으로 등급 구분 (회-초-파-보-주)

## 8. 밸런스 설계

### 8.1 수치 체계

**기본 공식:**

| 항목 | 공식 | 비고 |
|------|------|------|
| 스킬 데미지 | `BaseDmg × (1 + SkillLv × 0.2)` | 스킬 레벨당 20% 증가 |
| 적 체력 스케일링 | `BaseHP × (1 + WaveTime × 0.15)` | 경과 시간(분)당 15% 증가 |
| 경험치 요구량 | `10 + (Lv × 5)` | 레벨당 선형 증가 |
| 크리티컬 확률 | 기본 5% | 장비/패시브로 증가 |
| 크리티컬 배율 | 1.5x | 장비/패시브로 증가 |
| 방어력 감소 | `receivedDmg = rawDmg × (100 / (100 + DEF))` | DEF 100이면 데미지 50% 감소 |

**5분 세션 밸런스 목표:**

| 시점 | 플레이어 DPS | 적 밀도 | 체감 |
|------|------------|---------|------|
| 0~1분 | 낮음 (기본 무기만) | 낮음 | 여유, 탐색 |
| 1~2분 | 스킬 2~3개 | 중간 | 조합 실험 |
| 2~3분 | 스킬 4~5개 | 높음 | 위기감 → 진화 기대 |
| 3~4분 | 진화 스킬 발동 | 매우 높음 | 진화 후 쾌감 폭발 |
| 4~5분 | 풀빌드 | 최고 | 압도하거나 압도당하거나 |

### 8.2 경제 시스템

추후 확정. 장비 시스템 도입 시 함께 설계한다.

### 8.3 성장 곡선

**인게임 성장 (1회 세션 내):**
- Lv 1~5 (0~1분): 기본 스킬 습득 단계
- Lv 5~10 (1~2.5분): 빌드 방향 결정 단계
- Lv 10~15 (2.5~4분): 빌드 완성 및 진화 단계
- Lv 15~20 (4~5분): 풀빌드, 최종 전투

**메타 성장 (장비):**
- 장비 시스템 도입 후 설계. 기본 원칙: 장비 없이도 클리어 가능한 난이도 유지

### 8.4 난이도 밸런스 방침

- **첫 판 클리어율 목표**: 30~40% (적절한 도전감)
- **장비 수집 후 클리어율**: 70~80%
- **플레이어 스킬이 핵심**: 스킬 선택 + 이동 회피가 장비보다 중요
- **운 요소 범위**: 스킬 선택지 3택의 무작위성 (완전 운은 아님, 전략적 선택 가능)
